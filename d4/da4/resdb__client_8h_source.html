35// ResDBClient is used to send data to the server identified by ip:port or via
36// the provided socket. If SignatureVerifier is provided, data will be signed.
37// The client will retry max_retry_time_ if the connection or sned process fail.
38// For a message with a command, it will be set inside a Request message.
39// Each Request will be put into the network message ResDBMessage and signed.
40//
41// Message structre:
42// ResDBMessage:
43// Request:
44// cmd
45// user_message
46// signature
47//
48// For a raw message, it will be set inside a Network Message directly.
49// Message structre:
50// ResDBMessage:
51// user_message
52// signature
53//
54class ResDBClient {
55 public:
